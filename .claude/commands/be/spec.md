Đọc `docs/drafts/requirements.md` (nếu tồn tại) để hiểu yêu cầu sản phẩm.

Thực hiện Deep Research (nghiên cứu chuyên sâu) từ internet về best practices, data structure và tính năng cần thiết cho loại sản phẩm này. Sau đó:

1. Viết PRD (Product Requirements Document) chi tiết → lưu vào `docs/PRD.md`
2. Trong `CLAUDE.md`, thêm reference tới `docs/PRD.md`
3. Dựa trên PRD, tạo Implementation Plan với checkbox [ ] cho từng task nhỏ → lưu vào `docs/IMPLEMENTATION_PLAN.md`

Quy tắc:
- PRD phải cover: overview, target users, core features, out-of-scope, data models, API design
- IMPLEMENTATION_PLAN phải chia phase rõ ràng, mỗi phase có mục tiêu cụ thể
- CHƯA bắt đầu code — chỉ research và document
